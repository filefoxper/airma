# Concepts

There are some important concepts in this little system.

1. [model](concepts?id=model)
2. [instance](concepts?id=instance)
3. [store](concepts?id=instance)
4. [key](concepts?id=key)

## Model

Model is a function for state management. It receives a parameter as state, and returns an object or array as **instance**.

A model looks like:

```ts
// receives a state,
// the state can be any type.
function counting(state: number) {
    // returns an instance,
    // it can be a plain object or array.
    return {
        count: state,
        symbol: state < 0 ?'-':'+',
        ......,
        increase: ()=>state + 1,
        decrease: ()=>state - 1,
        add: (addition: number)=>state + addition,
        ......
    }
};
```

More examples:

1. count model:

```ts
// This is a count model.
// Receives a number value state.
// Returns a plain object instance.
const counter = (count: number)=>{
    return {
        count,
        isNegative: count < 0,
        increase(){
            return count + 1;
        },
        decrease(){
            return count - 1;
        }
    }
}
```

2. toggle model:

```ts
// This is a toggle model.
// Receives a boolean value state.
// Returns a tuple array instance.
const toggler = (visible:boolean):ToggleInstance =>[
    visible, 
    ()=>!visible
] as const;
```

## Instance

Instance is an plain object or array for rendering components. It is generated by **model**, and enabled by **store**. The methods of instance can rerender component by generating a new state for store refreshing. These methods are **action methods**.

```ts
import React from 'react';
import {useModel} from '@airma/react-state';

// This is a count model.
// Receives a number value state.
// Returns a plain object instance.
const counter = (count: number)=>{
    return {
        count,
        isNegative: count < 0,
        increase(){
            return count + 1;
        },
        decrease(){
            return count - 1;
        }
    }
}

const App = ()=>{
    // useModel creates a local store and returns the store instance.
    // The store instance is enabled.
    const instance = useModel(counter, 0);
    const {
        // Action method to generate next state and refresh instance in store.
        decrease, 
        // data for rendering
        count,
        // Action method to generate next state and refresh instance in store. 
        increase
    } = instance;
    return (
        <div>
            <button onClick={decrease}>-</button>
            <span>{count}</span>
            <button onClick={increase}>+</button>
        </div>
    );
}

// const localCounter = model(counter);
// ...
// const App = ()=>{
//    // same
//    const instance = localCounter.useModel(0);
//    ......
// }
```

The object produced from instance by **model(xxx).produce** method is a usage of instance, it will replace instance when use **useModel**, **useSignal**, **useSelector** API.

## Store

Store is created by model and initial state. It contains a state and an **instance** generated by **model** running. Only **instance** from store is enabled to provide action methods for dispatching new state action to store. Consider it as a **redux** action work.

```ts
import React from 'react';
import {model} from '@airma/react-state';

// Use model api to create a static store.
const counter = model((count: number)=>{
    return {
        count,
        isNegative: count < 0,
        increase(){
            return count + 1;
        },
        decrease(){
            return count - 1;
        }
    }
}).createStore(0); // initialize store static

const App = ()=>{
    // Use [store].useModel() can get an enabled store instance too.
    const instance = counter.useModel();
    const {
        // method to generate next state and refresh instance in store.
        decrease, 
        // data for rendering
        count,
        // method to generate next state and refresh instance in store. 
        increase
    } = instance;
    return (
        <div>
            <button onClick={decrease}>-</button>
            <span>{count}</span>
            <button onClick={increase}>+</button>
        </div>
    );
}
```

### How does it work?

1. Store creator calls model function with state to generate an instance for the new store.
2. Calling method of store instance can generate a new state. This new state is wrapped to be an action and be dispatched by method to store.
3. The store recalls model function with action state to refresh its instance, and then notify its subscribers.

Step 1:

```ts
import {useModel} from '@airma/react-state';

// API useModel contains a store creator.
// It creates a local store and generates store instance. 
// Now, the store instance is generated by initial state 0.
const {count, increase} = useModel(counter, 0);

function callIncrease(){
    // The method result is a new state,
    // it is wrapped to be an action, and dispatched to store. 
    increase(); 
    // state: 0 -> 1
}
......
```

Step 2:

```
Store:
1. Recalls `counter` with new action state to generate new instance;
2. Wrap new instance with state action to be a full action.
3. Send full action to middleWares(not opened yet), and finally to subscribers(useModel/useSignal/useSelector).
```

Step 3:

```ts
import {useModel} from '@airma/react-state';

// API useModel is a subscriber to its own store.
// When it receives full action from its store,
// it rerenders component.
const {count, increase} = useModel(counter, 0);

function callIncrease(){
    increase();
}
......
```

## Key

Key is a model wrapper for creating store. It is also an identifier for finding and subscribing the right store. The value of **key** is generating store in **react elements**. So the different **elements** generated by same **component** contains different stores with same **key**. 

```ts
import {
    createKey,
    provide,
    useModel,
    useSelector
} from '@airma/react-state';

// create key for generating store
const counterKey = createKey((count:number)=>{
    return {
        count,
        isNegative: count<0,
        increase(){
            return count+1;
        },
        decrease(){
            return count-1;
        }
    }
// give the key an initial state
}, 0);

const Increase = memo(()=>{
    // use key to subscribe store in Provider
    const increase = useSelector(counterKey, i => i.increase);
    return <button onClick={increase}>+</button>;
});
const Count = memo(()=>{
    // use key to subscribe store in Provider
    const {count} = useModel(counterKey);
    return <span>{count}</span>;
});
const Decrease = memo(()=>{
    // use key to subscribe store in Provider
    const decrease = useSelector(counterKey, i => i.decrease);
    return <button onClick={decrease}>-</button>;
});
// use HOC `provide` to generate store inside Provider elements.
const CounterComponent = provide(counterKey)(function Comp() {
    return (
        <div>
            <Increase/>
            <Count/>
            <Decrease/>
        </div>
    );
});

const App = ()=>{
    // This two elements `<CounterComponent/>` has same Component `CounterComponent`,
    // but they carry different stores. 
    // One store change can not affect another one.
    return (
        <div>
            <CounterComponent/>
            <CounterComponent/>
        </div>
    );
}
```

API [createKey](api?id=createkey) wraps a **model function** to be a unique `key` with or without an initial state.
